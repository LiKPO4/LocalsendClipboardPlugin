# LocalSend图片剪贴板插件 - 关键修复说明

## 问题分析

### 根本原因：Tkinter线程冲突

之前的版本（v1.0.0 ~ v1.0.3）都存在一个核心问题：**Tkinter窗口创建在线程中执行**

#### 问题代码（v1.0.3）：
```python
def _open_settings(self):
    import threading

    def open_settings_in_main_thread():
        def on_save(new_config: Config):
            self.config = new_config
            self.watcher.update_watch_dir(new_config.watch_dir)

        window = SettingsWindow(self.config, on_save=on_save)
        window.show()

    import tkinter as tk
    root = tk.Tk()
    root.withdraw()
    root.after(0, lambda: [open_settings_in_main_thread(), root.destroy()])
    root.mainloop()
```

**问题**：
1. ❌ pystray的菜单回调在**子线程**中执行
2. ❌ Tkinter只能在**主线程**中创建窗口和运行事件循环
3. ❌ 创建了两个Tkinter根窗口（root和SettingsWindow.root）
4. ❌ lambda表达式语法错误：`lambda: [expr1, expr2]` 返回列表，不是执行两个表达式

**后果**：
- 浏览按钮点击无响应
- 图片格式输入框显示空白
- 窗口无法正常关闭
- 整个设置窗口功能失效

## 解决方案：事件驱动架构（v1.0.4）

### 核心改进：线程解耦

```python
class TrayApp:
    def __init__(self):
        # ... 其他初始化
        self._show_settings_event = threading.Event()  # 事件标志
        self._settings_thread: Optional[threading.Thread] = None

    def _open_settings(self):
        """从托盘菜单调用（pystray线程）"""
        self._show_settings_event.set()  # 设置事件，触发窗口显示

    def _show_settings_window(self):
        """在独立线程中创建Tkinter窗口"""
        def on_save(new_config: Config):
            self.config = new_config
            self.watcher.update_watch_dir(new_config.watch_dir)

        window = SettingsWindow(self.config, on_save=on_save)
        window.show()  # Tkinter在自己的线程中运行

    def _settings_thread_target(self):
        """后台线程持续监听设置窗口请求"""
        while self._running:
            if self._show_settings_event.is_set():
                self._show_settings_event.clear()
                self._show_settings_window()
            time.sleep(0.1)

    def run(self):
        self.watcher.start()

        # 启动独立的后台线程监听设置窗口请求
        self._settings_thread = threading.Thread(
            target=self._settings_thread_target,
            daemon=True
        )
        self._settings_thread.start()

        self.tray.run()  # pystray在主线程运行
```

**工作流程**：
1. ✅ 用户点击"打开设置"（pystray线程）→ 设置事件标志
2. ✅ 后台线程检测到事件（每0.1秒轮询）→ 清除事件
3. ✅ 后台线程创建SettingsWindow（独立的Tkinter实例）
4. ✅ SettingsWindow.show() 运行事件循环
5. ✅ 用户关闭窗口 → 事件循环结束 → 后台线程继续监听

## 关键知识点

### Tkinter线程规则
- **必须**在主线程创建Tkinter根窗口
- **必须**在创建根窗口的线程中运行mainloop()
- 其他线程不能直接操作Tkinter控件

### pystray线程模型
- `tray.run()` 在主线程运行
- 菜单回调在**单独的线程**中执行
- 需要使用事件机制（如threading.Event）进行线程通信

### 事件驱动 vs 阻塞式
- **旧方案**：在回调中创建窗口并阻塞（错误）
- **新方案**：设置事件，让独立线程处理窗口（正确）
